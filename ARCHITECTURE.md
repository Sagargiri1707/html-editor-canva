## Overview

### Project Summary
A React-based in-place WYSIWYG HTML editor that enables visual editing of HTML content with a Canva-like smooth experience. The editor accepts HTML input, provides rich editing capabilities, and emits the modified HTML through onChange callbacks.

### Key Features
- âœ… In-place HTML editing with content preservation
- âœ… Full text formatting (bold, italic, underline, headings, lists, links, tables, code)
- âœ… Media management (images, videos, GIFs, SVGs) with replacement & upload
- âœ… Image editing (crop, resize, filters)
- âœ… Drag & drop element reordering
- âœ… Snap guides for alignment
- âœ… Floating contextual toolbar
- âœ… Undo/Redo history
- âœ… Theming support
- âœ… HTML sanitization

|---------|---------------|-------|

1. **Minimal Chrome**: Editor UI should not distract from content
2. **Contextual Tools**: Show only relevant tools for current selection
3. **Smooth Animations**: 60fps transitions, spring physics for natural feel
4. **Clear Feedback**: Visual indicators for all interactions
5. **Keyboard First**: Full keyboard navigation support

## Edge Cases & Error Handling

### Input Edge Cases

| Scenario | Handling |
|----------|----------|
| Empty HTML input | Show placeholder content or empty editable area |
| Malformed HTML | Parse with DOMParser (auto-corrects), log warning |
| Very large HTML (>100KB) | Show loading state, parse in chunks if needed |
| HTML with scripts | Strip all `<script>` tags during sanitization |
| Inline event handlers | Strip all `on*` attributes |
| External stylesheets | Preserve `<link>` tags, warn about external dependencies |
| Iframes to unknown origins | Preserve but show security warning |
| SVG with embedded scripts | Sanitize SVG content specifically |

### Editing Edge Cases

| Scenario | Handling |
|----------|----------|
| Selection spans multiple blocks | Apply formatting to each block separately |
| Empty selection formatting | Toggle format for insertion point |
| Nested list indent limit | Cap at 6 levels deep |
| Table cell merge conflicts | Prevent invalid merge operations |
| Delete last element | Prevent, show warning toast |
| Undo with no history | Disable undo button, no-op on shortcut |
| Rapid successive edits | Debounce history snapshots (300ms) |

### Media Edge Cases

| Scenario | Handling |
|----------|----------|
| Image load failure | Show placeholder with retry option |
| Video format unsupported | Show fallback message |
| Asset library empty | Show upload prompt |
| Upload failure | Show error toast with retry |
| Very large image upload | Client-side resize before upload |
| Crop results in tiny image | Set minimum crop size (50x50px) |

### UI/UX Edge Cases

| Scenario | Handling |
|----------|----------|
| Toolbar near viewport edge | Flip position to opposite side |
| Multiple elements selected | Show shared tools only |
| Drag outside editor bounds | Cancel drag operation |
| Snap guide conflicts | Prioritize closest snap point |
| Color picker near edge | Reposition picker within viewport |
| Long content in floating toolbar | Add horizontal scroll or dropdown |

## Technical Considerations

### Technology Stack (Recommended)

| Layer | Technology | Rationale |
|-------|------------|-----------|
| UI Framework | React 18+ | Modern hooks, concurrent features |
| State Management | Zustand or Jotai | Lightweight, good for local state |
| Styling | CSS Modules + CSS Variables | Theming support, no runtime cost |
| HTML Parsing | DOMParser (native) | Built-in, fast, reliable |
| Sanitization | DOMPurify | Battle-tested XSS prevention |
| Image Editing | react-image-crop + CSS filters | Lightweight, no canvas needed for filters |
| Drag & Drop | @dnd-kit/core | Modern, accessible, performant |
| Animations | Framer Motion | Smooth spring animations |
| Icons | Lucide React | Consistent, tree-shakeable |

### Performance Optimizations

1. **Virtual DOM Diffing**: Minimize re-renders with proper memoization
2. **Debounced onChange**: Don't emit on every keystroke (configurable delay)
3. **Lazy Modal Loading**: Code-split asset picker and image editor
4. **History Compression**: Store diffs instead of full snapshots for large docs
5. **Event Delegation**: Single event listener on editor root
6. **CSS Containment**: Use `contain: layout` on editor sections

### Accessibility

- Full keyboard navigation
- ARIA labels on all interactive elements
- Focus management in modals
- Screen reader announcements for state changes
- High contrast mode support
- Reduced motion preference respected

### Bundle Size Targets

| Package | Target Size |
|---------|-------------|
| Core editor | < 50KB gzipped |
| Asset picker | < 20KB gzipped (lazy) |
| Image editor | < 30KB gzipped (lazy) |
| Total (all features) | < 100KB gzipped |

## API Design

### Props Interface

```typescript
interface WysiwygEditorProps {
  // Required
  html: string;                              // Input HTML content
  onChange: (html: string) => void;          // Called on content change
  
  // Asset Management
  assets?: Asset[];                          // Pre-defined asset library
  onUpload?: (file: File) => Promise<Asset>; // Upload handler
  
  // Theming
  theme?: 'light' | 'dark' | ThemeConfig;    // Theme configuration
  
  // Features (all enabled by default)
  features?: {
    textFormatting?: boolean;
    headings?: boolean;
    lists?: boolean;
    links?: boolean;
    tables?: boolean;
    codeBlocks?: boolean;
    media?: boolean;
    imageEditing?: boolean;
    dragDrop?: boolean;
    snapGuides?: boolean;
  };
  
  // Behavior
  debounceMs?: number;                       // onChange debounce (default: 300)
  maxHistory?: number;                       // Undo stack size (default: 50)
  sanitizeOptions?: SanitizeConfig;          // DOMPurify options
  
  // Callbacks
  onSelect?: (selection: Selection) => void; // Selection change
  onError?: (error: EditorError) => void;    // Error handler
  
  // Styling
  className?: string;                        // Root element class
  style?: React.CSSProperties;               // Root element style
}

interface Asset {
  id: string;
  type: 'image' | 'video' | 'gif' | 'svg';
  url: string;
  thumbnail?: string;
  name: string;
  tags?: string[];
  width?: number;
  height?: number;
}

interface ThemeConfig {
  colors: {
    primary: string;
    accent: string;
    background: string;
    surface: string;
    text: string;
    textMuted: string;
    border: string;
    selection: string;
    snapGuide: string;
  };
  borderRadius: string;
  fontFamily: string;
}
```

```tsx
      html={html}
```
